# Текстовая стеганография в LLM

В этом проекте реализована текстовая стеганография на основе большой языковой модели (LLM): информация скрывается в тексте, генерируемом моделью GPT-2. Проект содержит два метода кодирования: кодирование Хаффмана (Huffman Coding) и кодирование фиксированной длины (Fixed Length Coding, FLC).

## Особенности

- Реализация стеганографии при генерации текста моделью GPT-2
- Поддержка двух методов кодирования:
  - Кодирование Хаффмана (Huffman Coding)
  - Кодирование фиксированной длины (Fixed Length Coding, FLC)
- Одновременная генерация стеганографического и обычного текста для сравнения

## Требования к окружению

- Python 3.8+
- PyTorch
- Transformers
- CUDA (опционально, для ускорения на GPU)

## Как использовать

1. Откройте Jupyter notebook:
```bash
jupyter notebook llm_stega.ipynb
```

2. Notebook содержит три основные части:
   - Реализация кодирования Хаффмана
   - Реализация кодирования фиксированной длины (FLC)
   - Генерация текста и стеганография

3. Генерация стеганографического текста:
   - Выберите метод кодирования (Хаффмана или FLC)
   - Задайте параметры (значение k для кодирования)
   - Запустите главную функцию с собственным промптом

Пример кода:
```python
# Инициализируем стеганографический обработчик
k = 2
handle = Huffman(k=2**k, bits=bits)
# Или используем FLC
# handle = FLC(k=k, bits=bits)

# Генерируем текст со стеганографией
prompt = "Hello，I'm"
stega_output, normal_output = generate_text_with_steganography(
    model, tokenizer, prompt, handle
)
```

## Выходные файлы

Программа создаёт два текстовых файла:
- `outputs-gpt2-stega.txt`: стеганографический текст
- `outputs-gpt2-normal.txt`: текст, сгенерированный обычным образом без стеганографических ограничений

## Важные замечания

- Модель GPT-2 скачается автоматически при первом запуске; в Китае можно предварительно скачать GPT-2 через зеркало https://hf-mirror.com, также можно использовать другие большие модели.

- Для лучшей производительности рекомендуется GPU с поддержкой CUDA
- Длина скрываемой информации влияет на длину генерируемого текста

## Принцип работы

1. Процесс встраивания (стеганография):
   - Преобразуем секретную информацию в двоичную последовательность
   - Генерируем текст с помощью модели GPT-2
   - С помощью метода кодирования встраиваем биты информации в генерируемый текст

2. Процесс декодирования:
   - Используем ту же модель GPT-2 и тот же контекст
   - С помощью метода декодирования извлекаем из текста скрытые биты информации
   - Преобразуем двоичную последовательность обратно в исходную информацию
